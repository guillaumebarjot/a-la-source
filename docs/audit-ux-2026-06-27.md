### Encore ouverts — liste ordonnée par priorité

#### P0 — Décisions produit actées (Guillaume, 27/06)

**D1 — Mode sombre : réactiver le toggle (doctrine non négociable)**
- `useUI.ts:16` : `toggleDarkMode: () => set(() => ({ darkMode: false }))` — le toggle est
  câblé à faux en dur, jamais appliqué.
- `App.tsx:35-37` : la classe `.dark` n'est jamais posée sur `.app`.
- Le code dark-safe est prêt et testé (tokens, garde-fous). Il suffit de rétablir la vraie
  bascule.
- Risque minimal : l'UI sombre existe et a été validée.

**D2 — Score DISPENSABLE : retirer le /100 de l'Inbox et le tri "Score animateur"**
- `Inbox.tsx:305-311` : jauge `/100` + `{s.score} / 100` affiché.
- `Ateliers.tsx:261` : `<option value="score">Score animateur (optionnel)</option>` —
  présenter un verdict chiffré juste sous le texte "on ne les note pas".
- Remplacer la jauge `/100` par `{nbFaits}/{JALONS_ORDRE.length} jalons` (stepper en clair,
  factuel, déjà visible juste en dessous).
- Retirer l'option "Score animateur" du sélecteur de tri.

**D3 — "bien qualifiée" → "prête à lire" (renommage doctrinal)**
- `Inbox.tsx:107` : texte `bien qualifiée` dans le paragraphe d'intro de la page.
- `Inbox.tsx:289` : classe CSS `.hub-carte--ok` (neutre) mais le texte `bien qualifiée`
  dans le commentaire et l'intro doit être renommé.
- Vérifier dans `inbox.css` si le label apparaît.

**D4 — EvaluationPanel : retirer le panneau chiffré (score Écho /40, Pédagogie /50)**
- `Sidebar.tsx:57-59` : `CollapsiblePanel title="Evaluation"` expose `EvaluationPanel`.
- `EvaluationPanel.tsx` : affiche `scoreTotal`, `moyEcho/40`, `moyPedagogie/50`.
- Décision : retirer le panneau entier ou le renommer "Évaluer" sans afficher les
  sous-scores (garder uniquement le nombre d'évaluations et un bouton "Évaluer" qui
  ouvre le formulaire ; supprimer l'affichage `scoreTotal`).

**D5 — Sidebar ouverte par défaut sur Mécanismes et Commentaires**
- `Sidebar.tsx:57-66` : tous les panneaux ont `defaultOpen={false}`.
- Décision : Mécanismes et Commentaires passent à `defaultOpen={true}`.

**D6 — Formats non aboutis : griser Arpentage + marquer "bientôt"**
- `Activites.tsx:73-80` : Arpentage présenté à égalité des autres formats opérationnels.
- Décision : badge "Bientôt" sur les cartes Arpentage (+ Parcours/quiz en création
  est opérationnel mais l'arpentage seul manque de finition diffusion).

**D7 — Fusionner Décryptage dans Dossier (même formulaire)**
- `Activites.tsx:52-59` : "Décryptage à chaud" = carte séparée mais pointe vers `/dossiers`
  (identique à Dossier). La distinction est déjà fonctionnelle (`a_chaud=1`) mais la
  carte Décryptage crée de la confusion.
- Décision : supprimer la carte "Décryptage à chaud" de la section Créer, garder "Dossier"
  avec une note "inclut le décryptage à chaud".

**D8 — Renommage des rubriques (Header nav) en langage d'usage**
- `Header.tsx:103-132` : Inbox, Veille, Sujets dans la nav.
- Décision tranchée par Guillaume : renommer en langage plus accueillant avec légende.
  Proposition : Inbox → "À trier", Veille → "À lire", sans toucher aux URLs.
- Sous-titre de rubrique dans le Header (légende du jargon).

**D9 — Recentrer la page Sujets sur la grille de thèmes**
- `Sujets.tsx:46-103` : H1 "À la source" + baseline éditoriale = doublon de l'Accueil.
- Décision : la page Sujets doit afficher d'emblée la grille de thèmes, pas un second
  accueil. L'intro doit être courte (une phrase) et centrée sur "par thème".

#### P1 — Chantiers encore ouverts (hors décisions)

**C1 — Sidebar : en-tête repliable non accessible au clavier**
- `Sidebar.tsx:18-28` : `<h3 onClick={() => setOpen(!open)}>` sans `role="button"`,
  sans `tabIndex={0}`, sans `onKeyDown`. Un utilisateur clavier ne peut pas replier les
  panneaux.
- Fix : ajouter `role="button"`, `tabIndex={0}`, `onKeyDown={(e) => e.key === 'Enter' && ...`.

**C2 — Suppression de tag sans confirmation**
- `TagsPanel.tsx:22-24` : `removeTag` poste la suppression directement sans `confirm()`.

**C3 — Suppression de commentaire sans confirmation**
- `CommentairesPanel.tsx` : à vérifier (non lu intégralement, mais signalé dans l'audit 22/06).

**C4 — Iframe "Source originale" blanche sans explication**
- `Lire.tsx:271-278` : l'iframe est rendue sans avertissement que la majorité des sites
  bloquent l'embed (X-Frame-Options). Seul un texte "Ouvrir dans un nouvel onglet" est
  proposé en fallback.
- Fix : ajouter un texte d'avertissement *avant* l'iframe ("La plupart des sites
  bloquent l'affichage en intégré. Si la page reste blanche, ouvrez-la dans un nouvel
  onglet.").

**C5 — Intercepteur 401/403 global manquant**
- `api/client.ts` : `request()` lève une erreur générique sur toute réponse `!ok`, sans
  détecter spécifiquement 401/403 (session expirée ou accès interdit).
- Fix : détecter `res.status === 401` pour déclencher une notification "Session expirée,
  rechargez la page" ou re-fetch de l'utilisateur.

**C6 — Carte canonique CSS manquante (~10 implémentations)**
- `globals.css` : `.source-card`, `.hub-carte`, `.activite-carte`, `.creer-carte`,
  `corpusdnd-card`... ~10 variantes de carte sans extraction d'une `.card` de base.
- Ce chantier est de refactoring CSS pur, sans régression fonctionnelle. À planifier
  séparément.

**C7 — Nom du média non cliquable vers sa fiche**
- `SourceCard.tsx:84` : `source.media_nom` affiché en `<span>`, jamais en lien.
- `Lire.tsx` : même constat dans le panneau Métadonnées.
- Fix : si `source.media_id` existe, rendre `<Link to={/observatoire/fiches?media=${id}}>`.

**C8 — Dédoublonnage catalogue mécanismes**
- `/apprendre` (`Mecanismes.tsx`) et `/observatoire/catalogue` (`Observatoire.tsx`) présentent
  tous deux le catalogue. Le CHANGELOG dit "désormais sous Observatoire" mais `/apprendre`
  existe encore.
- Décision à prendre : rediriger `/apprendre` → `/observatoire/catalogue`, ou conserver les
  deux avec des usages distincts (apprentissage vs référence).

**C9 — Jalon "Acceptée" pré-coché à la création (bug)**
- `Inbox.tsx:44` : `{ cle: 'accepte', label: 'Acceptee' }` affiché dans le stepper.
- À vérifier côté serveur : `sources.ts:276,484` indique `a_qualifier=0` à la création
  (source déjà en veille), ce qui coche le jalon "acceptée" pour une source neuve.
